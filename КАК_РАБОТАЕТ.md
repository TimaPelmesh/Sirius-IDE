# Как работает отображение и когда пересобирать

## 0. Важно по архитектуре

- **Окно всегда открывается по HTTP** (`http://127.0.0.1:PORT/renderer/index.html`). Fallback на `file://` убран: при неудаче запуска сервера приложение покажет ошибку и не откроет окно, чтобы не попадать в режим без воркеров Monaco.
- **Локальный сервер** поднимается при старте (свободный порт или 9876 при повторе). Без него редактор не работает.
- **Monaco** грузится через AMD loader с явным `baseUrl` (каталог страницы), иначе при загрузке по HTTP пути к `vs/editor/editor.main` и воркерам разрешались неверно. Воркеры берут тот же origin, что и страница.
- **Таймаут инициализации Monaco** — 25 с; при превышении показывается сообщение об ошибке.

## 1. Что вы видите в окне редактора

В приложении есть **два варианта** отображения кода:

| Вариант | Что это | Как выглядит |
|--------|---------|---------------|
| **Monaco** | Полноценный редактор (то, с чем работает логика подсветки) | Разноцветный текст: ключевые слова, строки, комментарии разного цвета |
| **Fallback** | Простое текстовое поле, если Monaco не удалось загрузить | Весь текст **одним цветом** (обычно белый на тёмной теме) |

Интерфейс и Monaco загружаются по **локальному HTTP** (`http://127.0.0.1:PORT`), а не по `file://`. Так веб-воркеры Monaco разрешены браузером, подсветка синтаксиса работает **полностью офлайн**, без интернета. Сервер поднимается при старте приложения и раздаёт файлы из папки проекта (renderer, node_modules).

## 2. Восстановление рабочей области (Ctrl+R и перезапуск)

**Что должно быть:** после перезагрузки страницы (Ctrl+R) или перезапуска приложения снова открыты ваша папка проекта, вкладки файлов и активный файл.

**Что было не так:** при восстановлении сначала вызывалось обновление дерева файлов и открытие файла, а корень проекта регистрировался в main-процессе позже. Из-за этого проверка путей (path validation) падала, дерево и файл не открывались, и вы оставались на начальном экране.

**Что сделано:** при восстановлении состояния теперь **сначала** вызывается `watchDir(root)` (и корень регистрируется в main), **потом** обновляется дерево и открывается активный файл. В конце восстановления принудительно скрывается приветственный экран и показывается область редактора.

Данные для восстановления берутся по порядку: из main-процесса (после Ctrl+R), из `localStorage`, при необходимости — из файла в `%APPDATA%/sirius-ide/state.json`.

## 3. Когда что запускать

| Действие | Команда | Зачем |
|----------|---------|--------|
| **Обычный запуск** | `npm start` | Собирает TypeScript (`npm run build`) и запускает приложение. Этого достаточно для работы. |
| **Пересборка после правок в `src/*.ts`** | `npm run build` | Компилирует TypeScript в `dist/`. Выполняется автоматически при `npm start`. |
| **Первая установка или после смены зависимостей** | `npm install` | Ставит пакеты и пересобирает native-модули (например, node-pty). Нужно после клонирования репозитория или изменения `package.json`. |
| **Только скомпилировать, не запуская** | `npm run build` | Нужно только если вы хотите обновить `dist/` без запуска приложения. |

Итого: для повседневной работы достаточно **`npm start`**. **`npm install`** — только после клонирования или изменения зависимостей. Файлы в **`renderer/`** (HTML, JS, CSS) при запуске через `npm start` подхватываются как есть, отдельная сборка для них не нужна.

# Сборка Sirius IDE

Документ описывает, как собирать и распространять проект: запуск из исходников, сборка без установщика (portable) и сборка с установщиком.

---

## Требования

- **Node.js** 18+ (рекомендуется 20 LTS)
- **npm** (идёт с Node.js)
- **Windows** — основной целевой режим (терминал через node-pty)

---

## 1. Запуск из исходников (режим разработки)

Так проект работает «как есть», без упаковки в .exe.

```bash
npm install
npm start
```

- `npm install` — ставит зависимости и пересобирает native-модули (node-pty).
- `npm start` — сначала выполняет `npm run build` (компиляция TypeScript в `dist/`), затем запускает Electron.

**Без установщика:** пользователь должен иметь установленный Node.js и выполнять эти команды в папке проекта. Удобно для разработки и для продвинутых пользователей.

---

## 2. Сборка проекта (компиляция)

```bash
npm run build
```

- Компилирует TypeScript из `src/` в JavaScript в `dist/`.
- Точка входа приложения: `dist/main.js` (указана в `package.json` → `"main"`).

После сборки запуск по-прежнему через `npm start` или `electron .` в корне проекта (Electron подхватывает `main` из package.json).

---

## 3. Упаковка в приложение (без установщика, portable)

Чтобы получить один исполняемый файл или папку с .exe, без установки через установщик, используют упаковщики Electron.

### Вариант A: electron-packager

Установка (один раз):

```bash
npm install --save-dev electron-packager
```

Добавить в `package.json` в `"scripts"`:

```json
"pack": "electron-packager . Sirius-IDE --platform=win32 --arch=x64 --out=release --overwrite"
```

Сборка:

```bash
npm run pack
```

Результат: папка `release/Sirius-IDE-win32-x64/` с `Sirius-IDE.exe` и всеми ресурсами. Папку можно копировать куда угодно и запускать .exe — **установщик не нужен**.

### Вариант B: electron-builder

Установка (один раз):

```bash
npm install --save-dev electron-builder
```

В `package.json` добавить секцию:

```json
"build": {
  "appId": "com.sirius.ide",
  "productName": "Sirius IDE",
  "directories": { "output": "release" },
  "win": {
    "target": "portable",
    "icon": "build/icon.ico"
  }
}
```

И скрипт:

```json
"dist": "npm run build && electron-builder --win"
```

Сборка:

```bash
npm run dist
```

Результат: в `release/` появится, например, `Sirius IDE 1.0.0.exe` (portable). Пользователь просто запускает этот файл — **без установщика**.

---

## 4. Сборка с установщиком

Тот же **electron-builder**, но с таргетом установщика (NSIS для Windows).

В `package.json` в секции `"build"`:

```json
"win": {
  "target": [ "nsis", "portable" ],
  "icon": "build/icon.ico"
},
"nsis": {
  "oneClick": false,
  "allowToChangeInstallationDirectory": true,
  "installerIcon": "build/icon.ico",
  "uninstallerIcon": "build/icon.ico"
}
```

Сборка:

```bash
npm run dist
```

В `release/` появятся:

- **Установщик** (например, `Sirius IDE Setup 1.0.0.exe`) — пользователь запускает, выбирает папку, ставит приложение, в меню Пуск появляется ярлык.
- **Portable** (если указан `"portable"` в `target`) — один .exe без установки.

---

## 5. Иконка и ресурсы

- Иконку вы подставляете сами (как и договорились).
- **Окно приложения:** иконка берётся из `build/icon.ico`, если файла нет — из `icon.ico` в корне проекта. Положите `icon.ico` в корень или в `build/` — при запуске окно и панель задач будут с вашей иконкой.
- **Сборка (electron-builder):** для установщика и portable поместите файл в `build/icon.ico` и в конфиге укажите `"icon": "build/icon.ico"` (и в `nsis` при необходимости).
- Если папки `build/` нет — создайте и положите туда иконку; в конфиге можно указать другой путь.

Без указания `icon` сборка всё равно выполнится, будет использована иконка Electron по умолчанию.

---

## 6. Краткая сводка

| Режим | Команда | Результат |
|-------|---------|-----------|
| Разработка / запуск из исходников | `npm install && npm start` | Запуск без упаковки, нужен Node.js |
| Только компиляция | `npm run build` | Обновление `dist/` |
| Portable (без установщика) | После настройки: `npm run pack` или `npm run dist` | Папка с .exe или один .exe |
| С установщиком | После настройки electron-builder + `nsis`: `npm run dist` | Setup.exe для установки в систему |

Текущий репозиторий настроен на **запуск из исходников** (`npm start`). Чтобы раздавать пользователям один .exe или установщик, нужно один раз добавить **electron-packager** или **electron-builder** и скрипты, как выше.
